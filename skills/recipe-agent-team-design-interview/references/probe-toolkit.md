# Probe Toolkit

Use these techniques during discovery. Pick the smallest set that resolves the current uncertainty; prefer two techniques when a phase has real ambiguity. Translate probe patterns into the requester's language.

## Phase Mapping

| Phase | Primary Techniques |
| --- | --- |
| Output Capture | forced choice, specificity demand, constraint inversion |
| Context & Story | story extraction, specificity demand, 5 Whys cascade |
| Specificity Drill | specificity demand, anti-reference |
| Trade-off Surface | trade-off surface, forced choice |
| Assumption Probe | assumption probe, 5 Whys cascade |
| Forced Choice & Priority | forced choice, trade-off surface |
| Failure Rehearsal | failure rehearsal, constraint inversion, 5 Whys cascade |

## Techniques

| Technique | Use When | Pattern |
| --- | --- | --- |
| 5 Whys cascade | Generic surface answer | Ask "Why does that matter?" up to five times. |
| Specificity demand | Abstract adjectives | Ask for one user doing one task, with no adjectives. |
| Anti-reference | Vague style direction | Ask for a similar bad solution and what fails. |
| Story extraction | Generic audience | Walk through one user's day before and after. |
| Forced choice | Multiple priorities claimed | Keep only one; rank the rest. |
| Assumption probe | Hidden invariants | List assumptions and which breaks the design if false. |
| Trade-off surface | Goals contradict | Name what is acceptable to lose to win the chosen goal. |
| Failure rehearsal | Confidence too high | Name three failure scenarios and earliest signals. |
| Constraint inversion | Timid decisions | Ask what changes if unconstrained, then why blocked now. |
